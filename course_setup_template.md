# 새 수업 프로젝트 세팅 템플릿

> 이 템플릿을 채워서 Claude에게 전달하면, 수업 프로젝트(CLAUDE.md, curriculum.md, README.md, notes/, labs/)를 자동으로 생성할 수 있습니다.

---

## 사용 방법

1. 아래 각 섹션의 `[작성]` 부분을 채운다
2. 예시를 참고하되, 자신의 수업에 맞게 수정한다
3. 완성된 문서를 Claude에게 전달한다 (맨 아래 **Claude 활용 가이드** 참고)

## 6. 콘텐츠 작성 규칙

> Claude와 소통할 때 혼동될 수 있는 용어나 표현이 있으면 여기에 정의합니다.

- [작성] (예: '추가해' = 현재 파일에 내용을 더 추가하라는 뜻, git add 아님)
- [작성] (예: 특정 용어의 해석 규칙)

**예시:**

- '추가해'가 콘텐츠/문서 맥락에서 사용되면 **'현재 파일에 내용을 더 추가하라'**는 뜻임. 'git add'가 아님.
- 명시적으로 git 워크플로우 중일 때만 git 명령어로 해석할 것.

- [작성] (예: 난이도는 항상 대상 수준에 맞출 것)

### 실습 과제 구성 템플릿

```

```

### 실습 과제 필수 포함 항목

- [작성] (예: 예상 결과 출력 예시)
- [작성] (예: 자주 하는 실수 및 해결 방법)
- [작성]

**예시:**

### 기본 규칙

- 교육 콘텐츠 작성 시 **해당 Day까지 배운 내용만** 포함할 것
- 이후 Day의 주제를 절대 포함하지 않음
- 난이도는 항상 대상(리눅스 완전 초보)에 맞출 것
- 범위가 불확실하면 먼저 어떤 Day/주제를 다룰지 질문할 것

### 실습 과제 구성 템플릿

```
Part 1: 실습 준비 (폴더/파일 생성)
Part 2: 명령어 연습 (개별 명령어 실행)
Part 3: 스크립트 작성 (복사/붙여넣기)
Part 4: 권한 설정 및 실행
Part 5: 결과 확인/저장
```

### 실습 과제 필수 포함 항목

- 예상 결과 출력 예시
- 자주 하는 실수 및 해결 방법
- 완료 확인 체크리스트
- Git 제출 가이드 (add, commit, push 명령어)

## 9. 생성할 프로젝트 구조

> Claude가 만들어야 할 폴더 및 파일 구조를 정의합니다.
> 기본 구조를 그대로 사용하거나, 필요에 맞게 수정하세요.

```
프로젝트명/
├── CLAUDE.md          # Claude 지시 파일 (수업 규칙, 가이드라인)
├── README.md          # 수업 개요 (교재 구성, 훈련 목표)
├── curriculum.md      # 전체 커리큘럼 (Day별 상세 계획)
├── notes/             # 일별 수업 노트
│   ├── day1.md
│   ├── day2.md
│   └── ...
└── labs/              # 실습 자료
    ├── 실습1.md
    ├── 실습2.md
    └── ...
```

### 파일별 용도 및 포함 내용

| 파일 | 용도 | 포함 내용 |
|------|------|----------|
| `CLAUDE.md` | Claude가 작업 시 참고하는 **프로젝트 규칙 파일** | 수업 정보, 콘텐츠 작성 가이드라인, 실습 과제 가이드라인, Git 워크플로우, 주요 파일 목록 |
| `README.md` | 저장소 방문자가 보는 **수업 소개 문서** | 교육 개요, 훈련 목표, 교재 구성, 주요 교수 계획 |
| `curriculum.md` | 전체 수업의 **상세 계획표** | Day별 시간 배분, 학습 목표, 실습 과제 목록, 최종 프로젝트, AI 활용 가이드 |
| `notes/dayN.md` | 각 Day의 **수업 노트** (수업 중 참고용) | 이론 설명, 명령어 사용법, 실습 예제, 주의사항, 트러블슈팅 |
| `labs/실습명.md` | **실습 과제 문서** (수강생 배포용) | 실습 단계, 예상 결과, 자주 하는 실수, 완료 체크리스트, 제출 가이드 |

### 추가 폴더/파일 (선택)

- [작성] (예: `assets/` - 이미지, 다이어그램 등)
- [작성] (예: `exams/` - 평가 자료)

> 위 템플릿을 채운 후, Claude에게 전달하여 프로젝트를 생성하는 방법입니다.

### Step 1: 프로젝트 초기 세팅

빈 Git 저장소를 만들고 Claude Code를 실행합니다.

```bash
mkdir 수업프로젝트명
cd 수업프로젝트명
git init
claude
```

### Step 2: Claude에게 프로젝트 생성 요청

아래 프롬프트를 복사하여 `[채운 템플릿]` 부분에 위에서 작성한 내용을 붙여넣습니다.
