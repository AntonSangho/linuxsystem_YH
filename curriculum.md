# 리눅스 시스템 수업 커리큘럼

## 과정 개요

| 항목 | 내용 |
|------|------|
| **훈련기간** | 50시간 (주 5일 × 5시간 × 2주) |
| **대상** | 개발자 취업 준비 비전공자/전공자 (리눅스 초보) |
| **교재** | 시스템 & 네트워크 우분투 리눅스 (한빛미디어) |

- 라즈베리파이 데비안 환경에서 ROS2를 운영하기 위한 능력
- AWS의 EC2 서버에 배포될 SpringBoot 서비스를 관리할 능력

# AI 조수(ChatGPT) 활용 가이드

## AI 도구 활용 원칙

| 구분 | 내용 |
|------|------|
| **역할** | 검색 대체가 아닌 "문제 해결 파트너" |
| **준비** | ChatGPT 무료 계정 회원가입 |
| **핵심** | 효과적인 질문법 습득 |

## AI에게 질문하는 기본 템플릿

```
[상황]
Ubuntu 22.04에서 Apache 웹 서버를 설치하고 있습니다.

[수행한 명령어]
sudo systemctl start apache2

[오류 메시지]
Job for apache2.service failed because the control process exited with error code.
See "systemctl status apache2.service" and "journalctl -xe" for details.

[질문]
이 오류의 원인과 해결 방법을 알려주세요.
```

## 실습 중 AI 활용 규칙

| 구분 | 내용 |
|------|------|
| **먼저 시도** | 오류 메시지 직접 읽기 → man 페이지 확인 → 그래도 모르면 AI 질문 |
| **복사할 것** | 정확한 오류 메시지, 실행한 명령어, OS 버전 |
| **하지 말 것** | AI 답변 무조건 복붙 → 반드시 이해 후 적용 |
| **기록하기** | 해결된 문제와 방법을 개인 노트에 정리 |

## 상황별 AI 질문 예시

### 1. 명령어를 모를 때
```
Ubuntu에서 현재 실행 중인 프로세스 중 메모리를
가장 많이 사용하는 것 5개를 보는 명령어는?
```

### 2. 오류 해결
```
[오류 메시지 전체 복사]

위 오류가 발생했습니다.
Ubuntu 22.04, MySQL 8.0 환경입니다.
원인과 해결 방법을 단계별로 알려주세요.
```

### 3. 개념 이해
```
리눅스에서 chmod 755와 chmod 644의 차이점을
실제 사용 예시와 함께 설명해주세요.
```

### 4. 설정 파일 작성
```
Apache에서 /var/www/mysite를 DocumentRoot로 하는
가상 호스트 설정 파일 예시를 보여주세요.
```

## 주의사항 

1. **AI는 틀릴 수 있다** - 항상 검증 필요
2. **보안 정보 노출 금지** - 실제 IP, 비밀번호 등 입력하지 않기
3. **학습 목적 유지** - 답만 얻지 말고 "왜 그런지" 이해하기
4. **의존하지 않기** - 기본기는 반드시 직접 익히기
