## 네트워크 연결 세부 정보 읽는 법

Windows에서 네트워크 연결 세부 정보를 열면 다음과 같은 정보가 나온다.

### 내부 네트워크 (LAN, 사설 네트워크) 에 해당하는 항목

| 항목 | 값 (예시) | 설명 |
|------|-----------|------|
| **IPv4 주소** | `192.168.0.95` | 내 컴퓨터에 부여된 **사설 IP 주소**. `192.168.x.x`로 시작하면 내부 네트워크 주소이다. |
| **IPv4 서브넷 마스크** | `255.255.255.0` | 같은 네트워크에 속하는 범위를 정해준다. 이 경우 `192.168.0.0` ~ `192.168.0.255` 까지가 같은 내부 네트워크이다. |
| **IPv4 기본 게이트웨이** | `192.168.0.1` | 공유기(라우터)의 내부 IP 주소. 외부 인터넷으로 나가는 **출입구** 역할을 한다. |
| **물리적 주소 (MAC)** | `E8-03-9A-6D-B6-7C` | 네트워크 카드의 고유 번호. 내부 네트워크에서 장치를 구별할 때 사용한다. |
| **링크-로컬 IPv6 주소** | `fe80::41e2:...` | `fe80`으로 시작하면 내부 네트워크에서만 쓰이는 IPv6 주소이다. |

### 외부 네트워크 (WAN, 공인 네트워크) 에 해당하는 항목

| 항목 | 값 (예시) | 설명 |
|------|-----------|------|
| **IPv4 DNS 서버** | `203.248.252.2` / `164.124.101.2` | 통신사(ISP)가 제공하는 **외부 DNS 서버** 주소이다. 도메인 이름(예: google.com)을 IP 주소로 변환해준다. |

> DNS 서버 주소가 `203.x.x.x`, `164.x.x.x` 처럼 `192.168`이 아닌 숫자로 시작하면 **외부(공인) IP 주소**이다.

### 그림으로 이해하기

```
[인터넷(외부)]
     |
     | ← 공인 IP (통신사가 부여, 이 화면에는 안 보임)
     |
 ┌───────────┐
 │  공유기     │ ← 게이트웨이: 192.168.0.1
 └───────────┘
     |
     | ← 사설 IP 대역: 192.168.0.x
     |
 ┌───┴───┬───────┬───────┐
 │       │       │       │
 내PC   노트북   폰    기타장치
 .95    .100    .101    ...
```

### 핵심 정리

- **192.168.x.x** → 내부(사설) 네트워크 주소. 같은 공유기에 연결된 장치들끼리만 통신 가능.
- **그 외 숫자로 시작하는 IP** (예: 203.x.x.x) → 외부(공인) 네트워크 주소. 인터넷에서 접근 가능.
- **게이트웨이 (192.168.0.1)** = 공유기. 내부 ↔ 외부를 연결해주는 다리 역할.
- **DNS 서버** = 외부에 있는 서버. 통신사가 제공하며, 도메인 이름을 IP로 바꿔준다.

### 사설 IP 주소 대역 (외울 필요는 없지만 참고)

| 대역 | 범위 |
|------|------|
| A클래스 | `10.0.0.0` ~ `10.255.255.255` |
| B클래스 | `172.16.0.0` ~ `172.31.255.255` |
| C클래스 | `192.168.0.0` ~ `192.168.255.255` ← 가정에서 가장 흔함 |

이 대역에 해당하면 **무조건 내부(사설) 네트워크**이다.

### 확인 명령어

```bash
# Windows에서 내 IP 확인
ipconfig

# Linux/Ubuntu에서 내 IP 확인
ip addr
# 또는
hostname -I

# 외부(공인) IP 확인 (인터넷 연결 필요)
curl ifconfig.me
```
